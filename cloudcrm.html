Objective

Design the Clean Professional CRM dashboard to manage the workflow, employees, and organization.

Client Goals

I have conducted the meeting with the Client, Project Manager, and Lead Developer to gather the basic ideas of the dashboard.

1. The Dashboard is multi purpose Organization management tools similar to sales force.

2. It will have two different logins Admin and User Logins. First Phase should be focused on the Admin Dashboard.

3. The Admin Login will have privilege to all modules of the dashboard based on their requirements.

4. The Client login will be assigned with the tools to manage workflow(Future Updates).

5. It will be role based module with multiple roles and privileges.

6. The Dashboard should have the module to manage employee, companies list under owner companies, and roles and privileges.

7. The Dashboard will be having the setting module to manage all the client company tools at the single page.

Insights After Brainstorming

1. We have decided to Design Two variants of the Dashboard Admin module and the Client Module.

2. Admin Module will have Company List, Add, Edit Module, and also the overall setting to provide tool web access to the screens(CRM, CRM, and SHOP)

3. Admin Module will have page to create edit or delete the Admins.

4. Admin Module will have Role and privilege page which provided access to the Admins screens.

5. There will be integration module for the admin panel so that we can include data from the third party application.

6. We will have a Global dashboard page with the Quantitative data of User login, The tool activity on the user logins, and Integration module. The list of employee and their relevant details. This page is common for both admin and user logins

7. The Admin will be navigating to the admin page thorough icon on the navigation bar( we can show it as the App or tool )

8. The Admin Dashboard will have Company list, and Client dashboard will have employee list.

9. The Client dashboard will have employee Management module and employee privilege module, and integration module.

10. We have planned to add notes module to take down notes of each progress and use it anywhere on the dashboard.

User Flow

I took a small note on what would be the features/functions that will be made by the user. Categories the features with the name and planned it into creation of each page
